# 《余烬荒原》背包与物品系统

状态：生效中（第一版）；适用范围：背包、物品、装备、消耗品、拾取与商人出售；更新：2026-07-11。

## 1. 体验目标

背包服务于“拾取战利品—判断价值—装备或处理—继续战斗”的短循环。玩家打开背包后应快速回答：拥有什么、哪件更强、哪些可以出售。整理行为提供控制感，但不得成为高频战斗的主要负担。

背包打开期间暂停战斗。重要状态至少通过图标/文字和颜色两种通道表达，不能仅依赖品质颜色。最终体验验收由用户在 GitHub Pages 完成。

## 2. 当前规则

- 容量为 5×4，共 20 个统一物品格；装备栏不占背包格。
- 物品类型为武器、护甲、消耗品。当前消耗品只有回血药。
- 武器提供攻击，护甲提供最大生命；同类装备同时只能穿戴一件。
- 回血药每瓶恢复 25 生命，每格最多 20 瓶；满血使用不消耗。
- 拾取堆叠物时先填充兼容且未满的堆叠，再占用空格。无可用容量时物品留在地面并显示反馈。
- 相同 `definitionId` 的堆叠物才可合并。装备不可堆叠。
- 拆分数量必须是整数，范围为 1 至原堆叠数量减 1，并要求至少一个空格。拆分产生新的物品实例 ID。
- 锁定物品不能标记废品、单件出售或批量出售。消耗品和已穿戴装备不可出售。
- 废品批售只处理已标记且未锁定的背包装备；原有“一键全部出售”处理所有未锁定背包装备。
- 物品和背包当前仅存在于单局内存，重置或刷新后清除。

## 3. 操作与反馈

| 操作 | 结果 | 反馈 |
| --- | --- | --- |
| `B` | 打开或关闭背包 | 遮罩面板；战斗暂停。`I` 不绑定背包 |
| 拖放 | 空格换位、占用格交换、同类堆叠合并 | 被拖物品跟随鼠标，释放后位置变化并显示文字确认 |
| 双击装备 | 与对应装备栏交换 | 装备文字、属性和拾取音效更新 |
| 双击药水 / `Q` | 消耗一瓶药水 | 治疗粒子、治疗音效、数量减少 |
| `Shift + 点击`堆叠 | 输入并拆分任意合法数量 | 成功或失败原因文字 |
| `Alt + 点击` | 锁定或解锁 | 锁图标和文字确认 |
| 右键装备 | 标记或取消废品 | 红色“废”角标和文字确认 |
| 悬停 | 查看名称、类型、品质、属性 | 装备显示相对当前穿戴的绿色/红色差值 |
| 筛选 | 全部、武器、护甲、药品 | 非目标格降低透明度，不改变真实位置 |
| 排序 | 按品质降序 | 原地重排，空格置后 |

新获得物品显示金色圆点，首次悬停后清除。稀有装备仍使用紫色边框；锁定、废品、数量分别使用独立角标，避免语义冲突。

## 4. 数据与模块边界

`src/inventory.js` 是规则与事务层，`src/game.js` 只负责输入映射、Canvas 绘制、战斗拾取和反馈播放。

物品实例的通用字段包括：`id`、`definitionId`、`type`、`name`、`rarity`、`color`、`locked`、`new`。装备实例增加攻击或生命属性及 `junk`；堆叠物增加 `quantity`、`stackMax` 和效果数据。

所有增加、移动、合并、拆分、消耗、装备、锁定、废品标记、排序和出售操作必须通过背包模块接口完成。装备栏与背包分离，批量出售不遍历装备栏。当前事务为同步内存操作；加入存档时必须增加版本迁移，加入联网经济时必须由服务端鉴权并原子执行。

`player.potions` 仅作为旧调用点的派生兼容属性，读取值来自真实背包堆叠，不保存独立数量。新代码应直接使用 `countItem`、`addItem` 和 `consumeItem`。

## 5. 异常与边界

- 非法拆分数量、原堆叠数量不足或没有空格时不改变状态，并返回明确原因。
- 向满堆叠拖放时不应吞掉物品；无法合并则保持原位置。
- 排序与筛选不改变装备栏、数量、锁定和废品状态。
- 背包满时地面掉落保留，避免奖励静默丢失。
- 快速重复输入不得产生负数量、重复实例或超过 `stackMax` 的堆叠。

## 6. 验证清单

- 18 瓶药水再拾取 7 瓶，应形成 20 和 5 两个堆叠。
- 将 15 瓶拆出任意 1–14 瓶；0、15、负数、小数和无空格均应拒绝。
- 将两个兼容堆叠拖到一起，应合并至上限，剩余数量保留。
- `Q` 与双击均只消耗一瓶；满血不消耗。
- 对比武器和护甲时，差值符号、颜色和最终装备属性一致。
- 锁定装备不能出售或标记废品；出售废品不影响普通、锁定、消耗品和已装备物品。
- 满包拾取药水或装备时，掉落继续留在地面。
- 筛选和排序后总物品数、堆叠总数及物品实例属性不变。

自动测试覆盖堆叠溢出、任意数量拆分、拖放合并、消耗、装备交换、锁定/废品出售和排序。Canvas 命中区域、视觉层级、音效和实际操作手感由 Pages 人工体验确认。

## 7. 后续范围

候选能力包括快捷栏、仓库、搜索、多选、键盘/手柄焦点导航、持久化和撤销。当前不引入重量、多格物品、耐久度，除非后续玩法证明整理或资源损耗本身能形成有价值的决策。

## 8. 演进历史

- 2026-07-11：初版背包将药水作为独立玩家计数，19 格只存装备；该方案已废弃。
- 2026-07-11：调研阶段曾因仅有一种堆叠物而暂缓任意数量拆分；用户明确要求后撤销该取舍。
- 2026-07-11：建立 20 格统一背包和真实药水堆叠，加入任意拆分、拖放、对比、排序、筛选、锁定、废品及批售规则。
