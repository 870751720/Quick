# 开始界面与本地存档

## 文档定位

- 用途：定义《代号：世界》当前开始界面、设置入口和继续游戏规则。
- 适用范围：Web 原型启动流程、浏览器本地存档、总音量设置。
- 维护状态：生效中。

## 开始界面

游戏载入后首先显示独立于 Canvas 世界的主菜单，标题为《代号：世界》，提供：

- **继续游戏**：读取最后一次有效本地存档；无存档时不显示该入口。
- **开始游戏**：重置角色和世界状态；存在存档时必须通过游戏内模态框确认覆盖，不得调用浏览器原生 `confirm`、`alert` 或 `prompt` 承担玩家流程。
- **设置**：打开设置面板，不进入游戏。

菜单使用原创像素场景图展示表面温暖正常的村庄、农田、商店与远处庄园。左侧暗部承载标题和菜单，右侧场景通过庄园的高位构图暗示压迫。背景使用缓慢推镜、阳光呼吸和低密度光尘形成分层动效；系统开启“减少动态效果”时停用非必要动画。进入游戏使用短淡出过渡。

菜单提供低音量循环氛围音乐、按钮悬停音和确认音。音乐与效果音均为 Web Audio 实时合成的原创程序音频，不依赖外部音频文件；受浏览器自动播放策略限制，音乐在玩家第一次指针交互后启动。进入游戏时音乐淡出，不能与战斗音效抢占听觉层级。

## 设置

当前提供总音量与静音。设置立即生效并独立持久化，不依赖游戏存档。浏览器音频仅在玩家主动点击开始或继续后解锁。

## 本地存档

- 存储介质：浏览器 `localStorage`。
- 存档键：`codename-world.save.v1`。
- 自动保存：游戏运行期间每 5 秒一次，并在页面关闭或刷新前再次保存。
- 当前内容：玩家位置、生命、金币、等级、经验、背包容量、背包物品和已装备物品。
- 不保存临时战斗现场，包括存活怪物、掉落物、粒子和浮动文字；继续游戏会以干净战斗现场恢复。
- 存档带显式版本号。结构不兼容时不得部分加载，应保持在开始界面并允许玩家开始新游戏。
- 当前为单机原型存档，不承诺跨浏览器、跨设备或清理站点数据后的恢复。

## 反馈与异常

- 无存档时不显示继续游戏入口，避免向首次玩家展示不可用操作。
- 覆盖已有进度前必须二次确认。确认框沿用主菜单的暗金视觉与程序音效，默认聚焦“覆盖并开始”；玩家可以选择“返回”、点击遮罩或按 `Esc` 取消，取消操作不得修改存档。
- 设置面板关闭后返回主菜单，不改变游戏进度。
- 存档解析失败时不启动游戏，避免用损坏数据污染运行状态。
