# 开始界面与本地存档

## 文档定位

- 用途：定义《代号：世界》当前开始界面、设置入口和继续游戏规则。
- 适用范围：Web 原型启动流程、浏览器本地存档、总音量设置。
- 维护状态：生效中。

## 开始界面

游戏载入后首先显示独立于 Canvas 世界的主菜单，标题为《代号：世界》，提供：

- **继续游戏**：读取最后一次有效本地存档；无存档时禁用并提示“暂无存档”。
- **开始游戏**：重置角色和世界状态；存在存档时必须确认覆盖。
- **设置**：打开设置面板，不进入游戏。

菜单视觉采用克制的黑暗奇幻风格，通过排版、低饱和色彩与光影建立气质，不用持续黑暗的环境画面替代叙事黑暗。进入游戏使用短淡出过渡。

## 设置

当前提供总音量与静音。设置立即生效并独立持久化，不依赖游戏存档。浏览器音频仅在玩家主动点击开始或继续后解锁。

## 本地存档

- 存储介质：浏览器 `localStorage`。
- 存档键：`codename-world.save.v1`。
- 自动保存：游戏运行期间每 5 秒一次，并在页面关闭或刷新前再次保存。
- 当前内容：玩家位置、生命、金币、等级、经验、背包容量、背包物品和已装备物品。
- 不保存临时战斗现场，包括存活怪物、掉落物、粒子和浮动文字；继续游戏会以干净战斗现场恢复。
- 存档带显式版本号。结构不兼容时不得部分加载，应保持在开始界面并允许玩家开始新游戏。
- 当前为单机原型存档，不承诺跨浏览器、跨设备或清理站点数据后的恢复。

## 反馈与异常

- 无存档的继续按钮必须具有禁用视觉，不允许点击后无反馈。
- 覆盖已有进度前必须二次确认。
- 设置面板关闭后返回主菜单，不改变游戏进度。
- 存档解析失败时不启动游戏，避免用损坏数据污染运行状态。
