# 主角房间系统

状态：生效中（原型）；更新：2026-07-12。

## 体验目标

场景 `101` 是新游戏的第一个可控空间。房间应表现贫穷、寒冷和长期缺乏维护，同时让玩家迅速读懂床、粮柜、壁炉、桌子与出口。美术首先服从角色一致性和可玩空间辨识度，不使用一张烘焙背景替代可交互场景结构。

## 组件与像素规范

- 房间使用 Tile 与独立 Sprite 拼接；墙地面、家具、角色、碰撞和交互点分别保留数据边界。
- 当前角色、家具和 16×16 基础 Tile 统一使用 `3×` 整数像素倍率。禁止单件家具使用独立小数倍率修补尺寸。
- 新增素材优先选择与 Shade/Puny、MiniWorld 同像素密度和轮廓语言的资产。缺失物件可以生成或绘制独立 Sprite，但不得直接生成整张运行时房间底图。
- Canvas 绘制关闭图像平滑。家具源图可以跨多个 16px 单元，但显示倍率必须一致。

## 布局与视觉层级

- 旧床位于左上生活区，是苏醒演出的第一焦点。
- 空粮柜和冷壁炉位于上墙，承担饥饿叙事；当前壁炉已经熄灭，不提供暖色环境光。
- 桌椅位于右侧，中央保留连续可行走空间；出口位于下墙中央，避免家具遮挡路径。
- 房间整体使用低饱和褐色和冷暗色。后续剧情重新点燃壁炉时，暖色只用于其附近的局部视觉引导，不能用大范围 Bloom 或平滑高分辨率纹理破坏像素风格。

## 物理、遮挡与交互

- 玩家移动判定使用脚点圆形碰撞体，不使用整张角色立绘矩形。
- 每件家具由同一个场景对象派生绘制范围、底座碰撞范围、Y 排序基准和调查热点，禁止另建不对应视觉位置的隐形判定。
- 家具只在接地部分阻挡移动；角色可与家具上半部视觉重叠，并根据脚点与家具底边进行前后排序。
- 出口范围由共享常量同时驱动墙体缺口、提示、调试显示与场景切换。
- 可调查物为旧床、空粮柜、冷壁炉和空碗。进入范围后显示低幅慢速浮动的 `E` 键提示；反馈不参与物理判定。

## 表现与降级

- 角色和家具绘制保持硬边像素，不启用插值。
- 后续壁炉环境光必须作为独立表现层，可在低性能或减少动效模式下关闭；关闭后不影响交互与寻路。
- 当前没有火焰动画和室内环境音。后续补充时必须独立于核心判定，并提供资源缺失时的静态降级。

## 验证

- 自动测试覆盖统一整数倍率、家具数据生成碰撞、玩家脚点、出口通道、主要家具间路径和八方向动画帧。
- 人工体验重点：角色与家具像素颗粒是否一致；角色经过床、桌椅前后时遮挡是否自然；碰撞是否贴合家具底座；中央到出口是否畅通；交互提示是否准确指向物件。
- 最终美术与玩法验收由用户在 GitHub Pages 版本完成。

## 演进历史

- 2026-07-12：废弃家具分别使用 `2.3×`、`3×`、`3.5×`、`4×` 的拼接方式，统一角色、家具和 Tile 为 `3×` 整数倍率；重排生活区、桌椅区和出口，并让交互点、碰撞与排序从场景对象数据派生。
