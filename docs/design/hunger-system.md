# 饥饿系统

## 文档定位

- 用途：定义玩家饥饿状态、时间变化、HUD 表现和存档规则。
- 适用范围：第一幕及后续生存驱动。
- 维护状态：生效中。

## 当前规则

饥饿值范围为 `0—100`，数值越高代表越饥饿。新游戏从 `72` 开始，游戏正常运行时每秒增加 `0.18`，最大为 `100`；GM 面板开启时暂停变化。当前原型只负责建立动机与状态反馈，尚不施加掉血、减速等惩罚，也没有进食恢复入口。

饥饿值属于玩家持久状态，必须进入本地存档。加载不含该字段的旧存档时使用新游戏初始值，不得导致存档失效。

## HUD 与反馈

HUD 常驻显示在游戏画面左上角，包含“饥饿”标签、百分比和由棕黄过渡至红色的进度条。达到 `65` 后进入警告状态，显示“腹中空空”，并用短促呼吸缩放、描边明暗和条内流光反复提示。动效不得遮挡操作区域或依赖声音传递必要信息；系统开启减少动态效果时，后续应提供静态降级。

## 验收

- 新游戏进入房间后立即看到高饥饿状态和循环警告动效；
- 数值随正常游戏时间增长且不超过 `100`；
- 刷新并继续游戏后饥饿值保持；
- GM 面板开启时数值暂停；
- 旧存档不含饥饿字段时仍可正常加载。
