# 第一幕：饥饿

## 文档定位

- 用途：定义新游戏开场过场、第一幕玩家动机和进入种田循环的叙事流程。
- 适用范围：新游戏过场、醒来场景、初次探索、第一次商店交互及偷盗铺垫。
- 维护状态：原型验证中。
- 边界：饥饿数值、食物效果、种植操作和偷盗判定应分别由对应系统文档定义；本文只规定第一幕内容与体验顺序。

## 第一幕目标

第一幕不使用欠租、催缴或固定期限驱动玩家。主角醒来后的直接问题是饥饿。玩家需要先获得食物，再逐渐发现种田是村庄允许穷人采用的生存方式。

饥饿同时承担三项功能：

1. 给刚进入世界的玩家一个无需解释即可理解的目标；
2. 自然引导购买种子、种植、浇水、收割和换取食物；
3. 为玩家趁人不注意拿走食物的行为提供情感动机，并为后续领悟偷盗铺垫。

## 新游戏过场

### 1. 黑暗中的感官

黑屏中先出现风声、鸟鸣和腹部收紧的声音。主角没有完整的穿越回忆，只感到陌生身体的寒冷与饥饿。

第一句内心文字：

> 肚子饿饿的。

这句话保留轻微的现代人口吻，与后续严肃世界形成反差，但不连续使用网络化或卖萌表达。

### 2. 在村边醒来

画面淡入明亮、正常的清晨村庄。主角躺在村边草地或废弃小屋中，身体虚弱，没有钱、食物和可立即收获的作物。玩家只获得少量移动与调查能力。

环境首先呈现食物：面包香气、早餐炊烟、窗边晾放的食物和商店货架。村庄并不故意折磨主角，其他人只把食物视为有明确归属的普通财产。

### 3. 第一次选择

玩家可以通过多种方式缓解第一次饥饿：

- 接受一次有限的免费食物或简单劳动换餐；
- 用仅有的小物件换取食物或种子；
- 在无人注意时拿走一份低价值食物；
- 调查环境找到被丢弃但仍可食用的食物。

第一顿饭不应将玩家卡死，也不立即进行道德评判。选择只影响少量对白、初始资源和隐藏的偷盗领悟记录。

### 4. 进入种田循环

玩家了解到快速生长作物可以在短时间内成熟。商店老板出售种子并收购作物；种植成为稳定解决饥饿的方法。第一项明确目标为：

> 想办法填饱肚子。

完成第一顿饭后，目标自然转为：

> 为下一顿饭做准备。

玩家由此进入购买种子、种植、浇水、收割和售卖的循环，而不是因为 NPC 发布“学习种田”的教程任务。

## 商店与战斗道具伏笔

商店是第一幕的高频交互点。老板持有从玩家原身体旁捡到的战斗觉醒道具，道具长期陈列在玩家可见位置，但第一幕不高亮、不解释功能。老板可以承认它是捡来的，却不在玩家刚醒来时主动归还。

第一次进店时，玩家的注意力应优先被食物和种子吸引。待生存压力缓解后，玩家才更可能注意到老板身后的异常道具。

## 偷盗铺垫

第一次可偷物应优先是能够立即食用的低价值食物。玩家需要明确知道它属于别人，并能观察物主视线或离开时机。

- 未被发现的拿取计入三次自然领悟偷盗的进度；
- 被发现时以制止、追回或关系变化为主，不在序章直接处死或永久锁死路线；
- 不显示“偷盗 1/3”等系统进度；
- 每次成功通过心跳、短暂停顿和内心文字反馈越界感；
- 玩家即使从不偷盗，也能通过种田、交易和关系路线获得战斗道具。

## 过场表现规则

- 首次完整过场目标时长 45–75 秒，之后尽快交还控制权；
- 任意确认键加速文字，长按跳过；跳过后仍正确设置第一幕状态；
- 第二次开始新游戏可以立即跳过；
- 饥饿声音保持克制，禁止高频循环或令人不适的咀嚼音；
- 村庄音乐温暖正常，不因饥饿切换为明显恐怖音乐；
- 过场和首个目标不展示战斗、GM 或版本信息。

## 当前原型范围

- 新游戏进入独立场景 `101`（主角房间），不再播放背景图加字幕的独立过场；
- 演出、角色和环境全部位于 Canvas 游戏场景内：黑场淡入、主角倒地、身体逐渐起身，随后直接交还移动控制；
- 演出中按空格或 Esc 可跳到可操作状态，跳过不会切换场景；
- 演出结束后仍停留在场景 `101`，玩家可调查空粮柜、空碗、冷壁炉和床；从右下方房门离开后进入场景 `100`；
- 继续游戏保存并恢复场景编号与场景演出状态，不重复已完成的苏醒过程；
- 旧战斗原型保留为场景 `900`，不会作为第一幕过场后的默认落点。

当前场景 `101` 使用原创俯视像素房间底图表现床、空粮柜、空碗、冷壁炉、农具、破窗和房门；场景 `100` 使用 CC0 Puny World 原生像素资源表现村边道路、农田、村屋、商店摊位和面包。苏醒过程、调查反馈和短期目标优先使用角色头顶气泡展示，仅在底部短暂显示必要操作提示。食物交互、饥饿数值、种植和偷盗尚未实现。

室内角色按房间家具尺度放大显示。床尾、桌沿、柜门和壁炉前缘作为独立前景遮挡层按角色纵深重绘；玩家进入调查点范围时，物体旁显示带呼吸动效的 `E` 图标和对象名称，离开范围自动隐藏。交互结果仍通过角色气泡反馈。

场景 `101` 不再使用整张房间背景图。墙地面与家具统一使用 octoshrimpy 基于 Shade MiniWorld/Puny 系列扩展的 `16x16 miniworld+` 室内资源，与主角保持相同的 16×16 网格、暖色板、轮廓宽度和俯视透视。每个对象记录图集裁切区、缩放与脚部占比；碰撞体和 `sortY` 从这些数据实时计算。

角色在房间内使用角色图集对应的八方向规则：移动输入实时更新朝向，方向行按反时针映射；行走仅在徒手帧 `1→2→3→2` 间往返，禁止进入第 4 列之后的持械动作；停止时回到对应方向的待机帧。起床旋转只作用于苏醒阶段，不得覆盖进入可操作状态后的朝向与移动动画。

房间调查只在玩家进入交互点有效距离后响应 `E`；距离外空按不得显示兜底气泡或产生伪反馈。交互提示只保留 `E` 键帽和下方名称，键帽上方不显示菱形、连接线或其他装饰。提示使用约 ±1.5px 的慢速上下浮动，不使用缩放呼吸。

房间底图包含画面外黑色留白，碰撞坐标以原画内实际房间边界 `x=184–1092、y=28–693` 为基准，不以 Canvas 四边作为墙体位置。场景原画或裁切方式变化时必须重新标定碰撞体、交互点、遮挡裁片与出口范围。

GM 模式默认开启，可按 F2 打开控制台。使用 `collision on`、`collision off` 或 `collision` 切换场景碰撞调试绘制：红色为静态碰撞矩形，青色为玩家碰撞圆，绿色虚线为出口触发区。正式体验可使用查询参数 `?gm=0` 关闭 GM 入口。

玩家在房间内的实体碰撞仅使用脚部圆形占地：圆心与脚下阴影中心一致，相对立绘中心向下偏移 34px，半径 18px。头部、躯干、武器和衣摆不参与场景阻挡；脚点同时作为家具遮挡 Y 排序依据，保证碰撞与视觉穿插语义一致。

## 待定事项

- 主角醒来的准确地点与第一位见到的 NPC；
- 第一顿饭的四条路径及具体资源差异；
- 快速生长作物的名称与食用方式；
- 饥饿是第一章叙事状态还是长期生存系统；
- 首次偷取食物的目标、物主和被发现后的反馈。
